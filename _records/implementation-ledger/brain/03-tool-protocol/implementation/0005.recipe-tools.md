# 0005 — Recipe Tools

## Purpose

Implement the three AI-callable recipe tools: `view_user_recipe`, `update_user_recipe`, `archive_user_recipe`.

Recipes are structured cooking knowledge — not a general skill, but a specialized record type with dedicated fields (`title`, `ingredients`, `steps`, `servings`, `cuisine`, `tags`, `source`, `notes`). The recipe index (title + description only) is injected into cooking session prompts. Full content is loaded on demand via `view_user_recipe`.

There is no `create_user_recipe` tool — recipes are created by the MiraSession DO at the end of cooking sessions via a direct DB write. The agent can only view, update (refine after cooking), and archive (retire) recipes.

`update_user_recipe` uses the same version archive pattern as skills — writes a snapshot to `recipe_versions` before overwriting. `archive_user_recipe` soft-deletes by setting `status = 'archived'`.

### 1. [NEW] [read.user.recipe.repository.ts](file:///Users/kibrukuture/Desktop/brioela/backend/src/agents/brain/_repositories/read.user.recipe.repository.ts)

```typescript
import { and, eq, getOne } from '@/database/drizzle/_database'
import { recipes } from '@/agents/brain/_schemas'
import type { BrainDatabase } from '@/agents/brain/_database'

export function readUserRecipe(database: BrainDatabase, id: string) {
	return getOne(
		database
			.select()
			.from(recipes)
			.where(eq(recipes.id, id)),
	)
}

export function readActiveUserRecipe(database: BrainDatabase, id: string) {
	return getOne(
		database
			.select()
			.from(recipes)
			.where(and(eq(recipes.id, id), eq(recipes.status, 'active'))),
	)
}
```
