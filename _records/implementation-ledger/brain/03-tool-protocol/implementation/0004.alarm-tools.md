# 0004 — Alarm Tools

## Purpose

Shipped `schedule_user_alarm` and `cancel_user_alarm` with the recipe-tool split pattern (`_schemas`, `_prompts`, `_executables`, `.tool.ts`).

Wake callbacks are injected from the Brain DO layer as `AlarmWakeCallbacks`:

- `scheduleAlarm(scheduledAtMs)` — sets DO wake to earliest pending `scheduled_at`
- `cancelAlarm()` — clears DO wake when no pending rows remain

Tool executables never call `this.ctx.storage.setAlarm()` directly.

## Session permissions (code: `get.brain.tools.ts`)

| Tool | chat | cooking | alarm | brain_maintenance | behavior_pattern_detection |
|---|---|---|---|---|---|
| `schedule_user_alarm` | yes | yes | no | yes | yes |
| `cancel_user_alarm` | yes | yes | no | no | no |

Alarm tools register only when `getBrainTools(..., wake)` receives `AlarmWakeCallbacks`.

## Schema alignment (complaints 003–006)

- Column: `scheduled_at` / `scheduledAt` (not `scheduled_for`)
- Column: `payload` (not `payloadJson`)
- Column: `triggering_session_id` / `triggeringSessionId` — migration `0007`
- Cancel writes: `cancelled_at`, `cancel_reason`; failures: `failure_reason` (not `failReason`)

## Dedup

`brain_maintenance_run` and `behavior_pattern_detection` reject a second pending row of the same `alarm_type`.
