# 0001 — Alarm Dispatch Handler

## Scope

Brain / Alarm system

## What

Create `dispatch.alarm.handler.ts` — reads pending `scheduled_alarms` rows ordered by `scheduledAt`, dispatches each to its typed case handler, marks alarm `fired`.

Handles the `session_watchdog` case: checks if session is still `active`, marks it `abandoned` with a brief outcomeSummary if so.

`brioela.brain.agent.ts` wires `alarm()` lifecycle method to call `dispatchAlarm`. Sub-agent alarm cases (`brain_maintenance`, `behavior_pattern_detection`) are added in `brain/sub-agents/` scope once those agents exist.

## Verification

- `bun run verify`
- Alarm row status = `fired` after dispatch
- Abandoned session: status = `abandoned`, endedAt set
