## Implemented

- Replaced limit-only Brain memory listing with keyset cursor pagination.
- Made `ListBrainMemoryEvents` require an explicit cursor: `null` for the first slice, then the returned cursor for continuation.
- Added `nextCursor` to the Brain memory listing contract.
- Ordered Brain memory events by `createdAtEpochMs desc, id desc` so the cursor is stable even when many events share the same timestamp.
- Added the Drizzle SQLite composite index on `brain_memory_events(created_at_epoch_ms, id)`.
- Regenerated the Brain Drizzle migration chain with snapshots so future Drizzle diffs are precise.
- Added `0001_memory_event_cursor.sql` as an index-only migration and regenerated the Brain migration manifest from Drizzle's journal.
- Updated the migration smoke call to use the explicit first-slice cursor.
- Added real lexicon entries for cursor pagination language.

## Rules Captured

- Brain memory streams do not use offset pagination.
- Large collection reads must be modeled as bounded ordered slices with a stable cursor.
- A cursor for non-unique ordering must include a unique tie-breaker column.
- Repository queries that introduce a cursor must have a matching Drizzle-owned index migration.
- Drizzle snapshots are part of the migration source of truth; SQL without matching snapshots is stale history.
