# 0004 — Session Compression Handler

## Scope

Brain / Session lifecycle

## What

Create `compress.session.handler.ts` — `checkCompressionNeeded(db, sessionId)` + `runCompression(db, env, sessionId, userId)`.

`checkCompressionNeeded`: reads session inputTokens + turnCount, returns true if past threshold (chat: 40 turns / 60k tokens; cooking: 80 turns / 100k tokens).

`runCompression`: collects all session turns, calls `generateObject` with `claude-haiku-4-5-20251001` to produce a 4-field summary (intent, accomplished, decisions, continuing), marks old session `compressed`, creates new child session linked via `parentSessionId`.

## Verification

- `bun run verify`
- Old session status = `compressed`, compressionSummary written
- New session row created with `parentSessionId` pointing to old session
