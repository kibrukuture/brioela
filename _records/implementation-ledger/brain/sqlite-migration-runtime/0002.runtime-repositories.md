## Implemented

- Added Brain migration run repository writes.
- Added Brain migration smoke result repository writes.
- Added Brain schema readiness reads.
- Added Brain migration lock repository functions backed by `agent_state`.
- Added Zod validation for the persisted migration lock JSON.
- Updated Brain migration startup to record run lifecycle states: `started`, `applied`, `smoke_passed`, and `failed`.
- Updated Brain migration startup to record smoke success and smoke/runtime failure rows.
- Updated readiness handling so normal readiness is proven by smoke success, not just Drizzle migrator return.
- Added lock handling where live locks produce `needs_retry` readiness without deleting another run's lock.
- Added migration error JSON formatting for readiness and run evidence.

## Rules Captured

- Drizzle still owns SQL application. Brioela records product readiness above Drizzle.
- Readiness becomes `migrating` only after this run owns the migration lock.
- A failed run releases the lock only when it owns that lock.
- Migration smoke evidence is stored in `brain_migration_smoke_results`.
- Until CI exposes a true deployment id, migration runtime rows use the shipped Drizzle migration tag as deterministic deployment identity.

## Not Implemented

- Control-plane rollout storage and policy evaluation.
- CI-provided deployment id binding.
- Retry attempt counting across historical runs.
- Workers runtime tests for fresh boot, repeated boot, smoke failure, and live-lock behavior.
