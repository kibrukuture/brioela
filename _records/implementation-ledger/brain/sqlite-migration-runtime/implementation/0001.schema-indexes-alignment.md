# 0001 - Schema Indexes Alignment

## Purpose

Align the index definitions in the Drizzle schemas with the index requirements stated in the database specifications. This ensures optimal query performance for index-based filters (active sessions, parent chains, constraint surfaces, and recipe stats) before any tools start querying them.

---

## Touched Files & Proposed Code Changes

### 1. `backend/src/agents/brain/_schema/session.schema.ts`

Add the missing indexes in the table's index array.

```typescript
// Target location: backend/src/agents/brain/_schema/session.schema.ts
// Add inside the returned array in the second argument function:

		index('sessions_type_status_started_at_index').on(table.sessionType, table.status, table.startedAt),
		index('sessions_parent_index').on(table.parentSessionId).where(sql`parent_session_id IS NOT NULL`),
		index('sessions_recipe_index').on(table.recipeId).where(sql`recipe_id IS NOT NULL`),
		index('sessions_started_at_index').on(table.startedAt),

```
