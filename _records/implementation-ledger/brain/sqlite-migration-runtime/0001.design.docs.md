## Implemented

- Added the dedicated Brain SQLite migration runtime guide.
- Documented why per-user SQLite migrations are distributed runtime events, not a single central database migration.
- Defined the migration manifest shape.
- Defined product migration tables:
  - `brain_schema_readiness`
  - `brain_migration_runs`
  - `brain_migration_smoke_results`
- Defined readiness states, including `ready`, `migrating`, `migration_failed`, `blocked_by_control_plane`, `needs_retry`, `read_only_degraded`, and `incompatible_code`.
- Defined per-Brain migration lock behavior.
- Defined safe and blocked migration timing.
- Defined control-plane rollout with canaries, percentages, kill switch, and failure thresholds.
- Defined expand / dual-write / backfill / verify / contract migration phases.
- Defined smoke test categories and hard rules.
- Defined forward-fix rollback policy.
- Updated Brain overview, Brain setup, Brain memory overview, SQLite schema guide, schema-version spec, and architecture backbone to reference the runtime.
- Corrected the `_migrations` folder design so role-suffixed files live under matching role folders.
