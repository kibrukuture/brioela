## Implemented

- `tools/brioela-reading-gate/` — new guard package, dayjs clock only (no native Date), zero JSON.parse, zero type assertions, fully typed.
- **Root daemon** (`run.gate.daemon.handler.ts`): refuses to run unless euid 0; serves a unix socket at `/var/brioela-gate/gate.sock` (chmod 666 so any agent can call it); routes `/read`, `/status`, `/check`.
- **Manifest** (`/var/brioela-gate/manifest/manifest.tsv`, folder mode 700 root-only): TSV lines `readAtMs · hash · bytes · workspaceRoot · file`, appended exclusively by the daemon. TSV chosen over JSONL so no JSON.parse boundary exists anywhere in the tool.
- **`/read` route**: daemon itself performs the OS read, rejects paths outside the workspace (realpath + prefix check) and paths containing tab/newline, hashes the bytes (sha256), appends the manifest entry, logs an event line, streams content back with `x-gate-hash` / `x-gate-bytes` / `x-gate-file` headers.
- **`/status` route**: per-file latest read with verdicts — `fresh` (inside 240-minute ttl AND current file hash matches) or `stale` (past ttl / file changed / file gone).
- **Client handlers**: `gate:read <file>` (streams content to stdout, proof summary to stderr; instructive error when the daemon is down: "Only the human can start it: sudo bun gate:up"), `gate:status`, `gate:up` (root check, folder modes, detached spawn, health poll, nice summary), `gate:down` (root check, SIGTERM via pid file).
- Root `package.json`: wired `gate:up`, `gate:down`, `gate:read`, `gate:status`.
- New lexicon slice `tools/brioela-lexicon-guard/_lexicon/tools/reading.gate.lexicon.constant.ts` (10 words: reading, gate, hash, fresh, socket, pid, url, content, bytes, serve) — wired into the tools aggregate through the human sudo-unlock ritual.
- Updated build-guide 14 guard tree to the lawful layout (name guard's locked `allowedUnderscoreFolders` has no `_daemon`/`_manifest`; everything conforms to `_helpers`/`_types` + root handlers).

## Rules Captured

- The daemon, not the client, performs the OS read — no self-reported reading exists.
- Daemon code conforms to guard law as found; locked guard configs are never edited by an agent, even when inconvenient.
- TSV over JSON for internal storage when it removes an unsafe parse boundary.
