## Implemented

**Phase A — verdict core (fail closed, signed green)**
- `runGuardBoard` runs all three guards into one `GateBoard`; `createBoardDiff` attributes added/fixed/kept versus the previous board.
- ed25519 keypair (`createGateKey`: private key root-only 0600 in `/var/brioela-gate/keys/`, public 644), `signGateText` / `checkGateSignature`.
- `/verdict` daemon route: runs the board, formats the dynamic verdict, and — only when clean and given a staged-diff hash — signs a green receipt bound to that exact hash (receipt log kept root-side).
- Signed heartbeat written every 5s; `checkGateHeartbeat` validates freshness + signature.
- `gate:verdict` client: fail closed (no daemon → BLOCKED with instructions), `--receipt` verifies the signature and the diff-hash binding client-side, `--hook` exits 2 for harness hooks.
- Pre-commit wall: `githooks/pre-commit` runs `verdict --receipt`; `gate:hooks:install` points `core.hooksPath` at it. Root `verify` script now IS `gate:verdict` — no daemon, no green, anywhere.

**Phase B — reactive watcher (fixes every defect found in the old one)**
- Daemon watches all eight workspace roots, reacts to EVERY event kind (folder create/rename/delete included — the exact class the old watcher silently dropped), retries missing roots every 30s, correct ignore globs, 250ms debounce, every event batch triggers a full board sweep streamed to subscribers.

**Phase C — feedback mouths (the loop hunts the AI)**
- `GUARD-RED.md` written at the workspace root while red — violations with inline fixes — deleted the instant the board is green. Ambient feedback every AI trips over in `ls`/`git status`.
- Claude Code PostToolUse hook (`.claude/settings.json`): every Edit/Write triggers `verdict --hook`; violations land in the AI's context seconds after the save. Proven live during this very build — the hook blocked and instructed its own builder repeatedly.
- Dynamic copy: delta attribution (`2 NEW · 3 fixed · 1 remain`), trend line of recent sweeps, and nearest-vocabulary suggestions (edit distance over the loaded lexicon: "did you mean: socket").
- `gate:watch`: live stream client with replay of the last 40 events and auto-reconnect.

**Phase D — self-defense**
- Tamper watch: any change under the guard/tooling folders (`tools/brioela-*`, `tools/scripts`) writes a root-owned tamper record; the board freezes red (HTTP 423) until the human runs `gate:tamper:clear` (root-only — the file lives in a root-owned folder, agents cannot unlink it).
- launchd LaunchDaemon with KeepAlive: `gate:up` writes `/Library/LaunchDaemons/com.brioela.reading-gate.plist` and bootstraps it — kill the daemon and launchd resurrects it. `gate:down` bootouts.
- Baselines: protected by the existing schg locks + tamper watch (chown approach rejected — schg already blocks edits, tamper watch catches unlock-window doctoring).
- Lock scripts hardened separately this session: locked = no edit/delete/rename (schg), unlocked = edit-only via macOS ACL `deny delete` on the file (proven live: edit ✓, rm ✗, mv ✗) — no directory ownership games, no side effects on file creation.

**Phase E — the guard attacked like an enemy**
- 14 unit tests green: signature round trip + forged-text rejection, board diff attribution, word distance/suggestions, manifest round trip, freshness.
- `gate:smoke`: live drill against the running daemon — socket alive, heartbeat signed+fresh, manifest forgery attempt (must EACCES), verdict round trip, violation injection (bad folder → board red → GUARD-RED.md raised → cleanup → green → flag lowered).

## Rules Captured

- Fail closed: absence of a live, signed verdict is red, never green.
- Green is cryptographic: a receipt exists only if the root daemon witnessed a clean board for that exact staged diff.
- The guard defends itself: tamper freezes everything until the human ritual.
- One verdict, many mouths: pre-commit, verify, harness hooks, GUARD-RED.md, and the watch terminal all replay the same daemon judgment.
