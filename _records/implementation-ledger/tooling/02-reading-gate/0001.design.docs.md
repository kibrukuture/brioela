## Implemented

- Wrote `build-guide/02-coding-standards/14-reading-gate.md` — the Reading Gate doctrine: a fourth guard (`tools/brioela-reading-gate/`) that hard-blocks commits unless the producing agent proves fresh reading of all Required Reading via `gate:read` manifest entries, passes mandatory two-layer attestation (Layer A: parsed ground-truth diffs for schemas/exports/enums; Layer B: randomized excerpt challenges for all text), and keeps the diff inside the ledger entry's Touched Files. Enforcement at git pre-commit + `bun run verify`. Live observability via `gate:watch` / `gate:status`.
- Wrote `build-guide/02-coding-standards/15-agent-loop-orchestration.md` — the loop doctrine: ledger entries as the work queue, ledger-author agent (gated), one implementer per git worktree (gated), fresh-context reviewer agent writing `REVIEW.md` (gated), orchestrator daemon with merge tracker, assignment board (`loop:status`), `INBOX.md` feedback router, and stall detector. Human role reduced to review + merge only.

## Rules Captured

- No file locks for daily-work files — `chflags` stays reserved for permanent, rarely-touched areas. The wall is at the landing points (commit/verify), never at write time.
- The gate trusts no instruction file, no memory, no agent — only bytes, hashes, and exit codes. CLAUDE.md is enforcement-irrelevant.
- Attestation is mandatory, not optional, for every pass.
- Reads expire (freshness window / hash change). Cached or memorized knowledge can never pass.
- Writing a ledger entry is itself gated work — entries cannot cite unread documents or unattested tables.
- No two active assignments may overlap in Touched Files.
- Enforcement must be agent/IDE/CLI agnostic: git + CLI + exit codes only.
