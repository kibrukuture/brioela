# Design System — Overview

## What This Folder Covers
The complete visual and interaction language for Brioela. Typography, color, spacing, motion, Skia layers, generative UI, haptics, texture, component variants, icon system, loading and empty states, and design philosophy. Every feature UI is built from this system — nothing is designed inline in a feature file.

## Key Decisions

- **Light mode first.** Dark mode is fully accommodated via NativeWind `dark:` variants but every decision is made light-first.
- **Warm cream base** (`#F8F6F2`) not clinical white. Warm near-black (`#0E0C10`) not cold black in dark mode.
- **NativeWind exclusively.** No StyleSheet anywhere. All styling via Tailwind class names.
- **CVA for all components.** Type-safe variant management — no ad-hoc conditional class strings.
- **Phosphor Icons.** Six-weight axis matches the typographic system. Duotone never used.
- **No skeleton screens. No PNG textures. No ActivityIndicator.** The design system provides creative replacements for all of these.
- **Generative UI implementation approach TBD** — pattern is decided (decision object + registry + Zod), delivery method (library vs custom build) decided at build time.
- **Haptics are a design layer** — semantic tokens, paired with motion configs, managed centrally.
- **Skia grain texture** — generated by SkSL shader, not PNG. Renders at native pixel density, static per session.
