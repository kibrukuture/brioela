# Passport — Feature Integration

## What This File Covers

How Passport integrates with Menu Scanning, Mesa, Bela, Medical Conditions, Kids Mode, Travel, Verified Profiles, and Viral Sharing.

---

## Menu Scanning

After menu scan, Passport can gather yellow waiter questions and red avoid rules into a staff-facing handoff.

Example action:

```text
Create Passport for staff
```

---

## Mesa

Mesa Passport uses Food Audience.

Rules:

- generic member wording by default
- no private member names by default
- include only relevant table-level rules
- expires after meal/session by default

---

## Bela

Bela Passport helps shoppers substitute correctly.

Includes:

- hard avoid rules
- preferred substitution rules
- scan-before-buy instructions
- uncertainty instruction: ask/scan before replacing

---

## Medical Conditions

Medical condition details are minimized into food rules.

Examples:

- Warfarin → vitamin K consistency note
- celiac → gluten/shared fryer avoidance
- pregnancy → raw/unpasteurized/high-mercury avoidance

No diagnosis or treatment language.

---

## Kids Mode

Caregiver/school Passport can be generated by parent.

Rules:

- no child name by default
- parent controls sharing
- simple snack/meal instructions
- no child account required

---

## Travel

Travel Passport uses destination language and local food-label context when available.

It should be accessible offline once generated.

---

## Verified Profiles

Practitioner annotations can be included only if:

- user granted practitioner access
- annotation is active
- user approves inclusion in Passport preview

---

## Viral Sharing Boundary

Passport is not Discovery Cards.

Do not optimize Passport for virality. Optimize it for correctness, clarity, privacy, and real-world usefulness.
