# Menu Scanning — Dish Verdicts

## What This File Covers

How each parsed dish becomes green, yellow, or red using the user's constraint profile from the Brain. This file owns the safety rules for dish-level verdicts.

---

## Constraint Profile Source

Menu scanning pulls the user's active constraint profile from the Brain DO. It does not re-derive allergies or dietary rules from scratch per scan.

Required profile inputs:

- Confirmed hard allergies.
- Intolerances.
- Dietary identity such as vegan, vegetarian, halal, kosher, gluten-free, diabetic, pregnant, or low-FODMAP.
- Medical-condition food watchlists when available.
- Soft dislikes and preferences.

The profile must distinguish hard constraints from soft preferences. A dislike can lower ranking or add context, but it cannot produce the same red safety state as a confirmed allergy.

---

## Verdict Values

Use three user-facing verdicts:

| Verdict | Meaning | Rule |
|---|---|---|
| Green | likely OK for this user | no known conflict and enough dish detail to be useful |
| Yellow | ask before ordering | possible conflict, missing ingredient detail, shared-prep concern, or low extraction confidence |
| Red | avoid | confirmed hard allergy or hard dietary/medical conflict is visible in the dish text |

Internal values can be `safe`, `caution`, and `avoid`, but the UI language should be green/yellow/red.

---

## Red Rules

Return red when a parsed dish visibly conflicts with a hard constraint.

Examples:

- Confirmed peanut allergy + dish name or description contains peanut.
- Gluten-free hard constraint + dish explicitly lists wheat noodles.
- Vegan dietary identity + dish explicitly contains chicken, beef, dairy, egg, or fish.
- Pregnancy watchlist + dish explicitly says raw fish or unpasteurized cheese.

Red requires visible evidence from the menu text or a known deterministic synonym/category mapping. Do not infer red from cuisine alone.

---

## Yellow Rules

Yellow is the default for uncertainty. Unknown means ask, not assume safe.

Return yellow when:

- Ingredient detail is missing or too sparse.
- Vision extraction or parser confidence is low enough to affect safety.
- A cooking method creates likely cross-contact risk, such as shared fryer for gluten, shellfish, or peanuts.
- A dish category often depends on hidden ingredients, such as sauces, marinades, dressings, broths, batters, or desserts.
- A soft conflict exists but not enough evidence for red.
- Community notes mention uncertainty or inconsistent handling for the place.

Yellow must always include one exact question generated by `04-waiter-questions.md`.

---

## Green Rules

Return green only when there is no known conflict and the dish has enough detail to be useful.

Green does not mean guaranteed safe. It means Brioela found no visible conflict in the menu content and the available profile.

Green copy should avoid absolute language:

- Use: "No visible conflict found."
- Avoid: "Safe for you."
- Avoid: "Allergy-safe."

For serious allergies, the detail screen should still show a short caveat that restaurant preparation can vary.

---

## Ranking

The list order should help users find practical options quickly:

1. Green dishes with strong detail and no preference conflict.
2. Green dishes with minor soft dislikes ranked lower.
3. Yellow dishes grouped by askability, with clearer questions first.
4. Red dishes at the bottom or hidden behind an "avoid" section depending on UI design.

Ranking never changes a verdict. A disliked ingredient can lower a green dish, but it cannot make a hard-allergy red dish less severe.

---

## Result Shape

```typescript
type MenuDishVerdict = {
  dishId: string
  dishName: string
  verdict: "green" | "yellow" | "red"
  reason: string
  matchedConstraints: Array<{
    constraintType: "hard_allergy" | "intolerance" | "dislike" | "dietary_identity" | "medical_watchlist"
    entityValue: string
    severity: "hard" | "soft"
  }>
  waiterQuestion: string | null
  confidence: number
}
```

`waiterQuestion` is required for yellow and optional/null for green and red.
