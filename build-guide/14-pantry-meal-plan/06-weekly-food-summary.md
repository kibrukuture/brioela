# Pantry And Meal Plan — Weekly Food Summary

## What This File Covers

Weekly food summary generation from scans, receipts, recipes, constraints, and sessions.

## Source Specs

- `brioela-specs/16-weekly-food-summary.md`

## Output

- one-line summary
- two to four key observations
- optional one action recommendation

## Data Sources

- scan events
- receipt spend
- recipe activity
- constraint matches
- Ground interactions
- cooking sessions

## Data Model

### `weekly_summary`

- `user_id`
- `week_start`
- `summary_json`
- `generated_at`
- `delivered_at`

## Generation

Generated by Orchestrator alarm or durable workflow.

Avoid overconfident causal claims.

## Delivery

Delivery follows `12-notifications` rules:

- medium priority
- one push max per day
- Sunday morning preferred
- in-app fallback if suppressed
