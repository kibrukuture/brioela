# Brioela Generative Grammar — Surface Integration

## What This File Covers

Which Brioela surfaces can use the grammar, which node families they can use, and what remains static.

---

## Strong Yes Surfaces

- scan result secondary explanation
- recipe card emphasis
- cooking session opener
- Food Time Machine
- weekly summary
- Mesa compatibility explanation
- menu scan summary
- Kids learning explanation container
- Discovery Cards
- savings/receipt story
- verified creator recipe attribution

---

## Strong No Surfaces

- hard allergy block
- medical condition hard flag
- recall alert
- payment/checkout
- permissions
- account/security settings
- practitioner/client consent
- destructive deletes
- child safety override
- Ground submission/authenticity gate

---

## Scanner

Allowed:

- secondary explanation framing
- ingredient insight layout
- first-time/familiar context
- swap suggestion presentation

Never generative:

- verdict level
- hard allergy block
- condition hard flag

---

## Mesa

Allowed:

- table fit summary
- member compatibility visualization
- substitution explanation

Never generative:

- per-member hard conflict calculation
- member names beyond approved labels
- medical/member privacy redactions

---

## Discovery Cards

Discovery Cards can use the grammar as a layout source, but final share output is a static image/artifact after privacy scrub.

---

## Cooking

Allowed:

- session opener
- recipe step rail emphasis
- technique note framing
- low-energy/simple mode framing

Never generative:

- timer controls
- safety interruptions
- tool confirmations for destructive changes
